# Changelog 2026-05-10

Stage 8 of the KPU SKU + ProcessNode + CoolingSolution plan: silicon
floorplanner + GEOMETRY-category validators. Two complementary floorplan
views ship in this PR -- a circuit-class binning (what silicon library was
built) and an architectural-role binning (compute / memory / memory_controller
/ IO / control) -- both derived from the same `silicon_bin`. The
architectural view is the new default for `cli/show_floorplan.py`; the
circuit view remains accessible via `--view circuit`.

This work merges in PR #152 (squash commit `48cb7e7`) and closes the Stage 8a
+ 8b items in `docs/designs/kpu-sku-and-process-node-plan.md`. Stage 8c
deferred items (DVFS feasibility, memory-bandwidth headroom, yield risk,
SVG / HTML viz, NoC routability validator, calibrate floorplan thresholds
against measured silicon) remain on the docket.

## Added

- **`silicon_floorplan.py` -- circuit-class view** at
  `src/graphs/hardware/silicon_floorplan.py`. `derive_kpu_floorplan(sku, node)`
  bins blocks by silicon library (HP_LOGIC / SRAM / ANALOG / IO),
  derives per-tile-class pitches, and lays out the mesh with a unified
  pitch + PHY strip + IO ring + control corner. Emits `Floorplan`,
  `FloorplanBlock`, `TilePitch`. (PR #152, ~570 lines.)
- **`silicon_floorplan.py` -- architectural-role view** in the same module.
  `derive_kpu_architectural_floorplan(sku, node)` re-bins by what the
  architect calls them: COMPUTE = PE + L1 + L2, MEMORY = L3,
  MEMORY_CONTROLLER = DRAM PHY, IO_PAD, CONTROL. Layout is a true 2D
  checkerboard (`(r+c)%2 == 0` -> COMPUTE, else MEMORY) so every interior
  compute tile has 4 memory neighbours. Memory controllers are
  rectangular blocks sized by the SKU's `memory.memory_type` +
  per-channel I/O width (LPDDR narrow stripes ~5:1 aspect, HBM squarer
  microbump arrays ~1.5:1) with count = `memory.memory_controllers`.
  Off-die DRAM channels drawn just outside the die boundary, paired 1:1
  with their MCs. Per-class summaries report whitespace contribution;
  `WhatIfDieEstimate` enumerates the die area if every compute tile were
  one class. Anchors MCs to die edges (not mesh edges) so they hug the
  IO ring even when extra_x / extra_y > 0. (PR #152, ~940 lines.)
- **`cli/show_floorplan.py`** ASCII renderer with channel labels outside
  the die. Auto-detects `--output` extension across JSON / CSV / MD /
  text (matches the `validate_sku.py` pattern); `--width` validated as
  positive integer; `--view {architectural,circuit}` flag selects which
  derivation to use; `--list` enumerates catalog SKU ids. Glyph map for
  architectural (`C`/`M`/`D`/`:`/`*` + `=`/`|` for off-die channels) and
  circuit (`H`/`B`/`#`/`~`/`:`). (PR #152, 750 lines.)
- **5 GEOMETRY validators** at
  `src/graphs/hardware/sku_validators/validators/geometry.py`:
  - `floorplan_pitch_match` -- compute-tile-class pitch deltas
    (within-class concern from Stage 8a)
  - `floorplan_within_die_envelope` -- declared `die.die_size_mm2` vs
    derived floorplan area
  - `floorplan_aspect_ratio` -- die aspect bounds for
    routing/packaging
  - `floorplan_compute_memory_pitch_match` -- the **primary KPU
    checkerboard concern**: compute pitch vs memory pitch
  - `floorplan_whitespace_fraction` -- top-level "die efficiency" alarm;
    finding message names the worst-contributor compute class and the
    smallest-die what-if alternative
  - Validator count: 9 -> 14. WARN-max thresholds during heuristic-v1
    so all 4 catalog SKUs land below ERROR. (PR #152, 421 lines.)
- **`tests/hardware/test_silicon_floorplan.py`** -- 74 tests parametrized
  over the 4-SKU catalog. Locks in: structural correctness (compute tile
  count = mesh, no overlaps, every tile carries a class), unified-pitch
  invariants, geometry validator registration + firing, MC count =
  `memory.memory_controllers`, LPDDR aspect >= 3.0, HBM aspect < LPDDR,
  DRAM channels strictly outside die, MC area independent of mesh size,
  true 2D checkerboard (interior C has 4 M neighbours), what-if
  estimates, CLI smoke tests for both views + JSON output. (680 lines.)
- **Docs**: `docs/designs/kpu-sku-and-process-node-plan.md` updated --
  Stage 8a + 8b marked done, Stage 8c open items enumerated.

## Changed

- **Validator count tracker** in `tests/hardware/test_sku_validators_phase2cd.py`
  (renamed `test_total_validator_count_after_phase2cd_and_stage8`):
  9 -> 14. (Phase 2b 7 + Phase 2c+d 2 + Stage 8a 3 + Stage 8b 2.)
- **`_arch_place_io_ring` and circuit-class `_place_io_ring`** now use
  `_IO_RING_THICKNESS_MM` (= 0.30 mm) unconditionally. Earlier code
  scaled thickness by `io_area / perimeter`; for SKUs with small `io_pads`
  area the ring shrank below one ASCII cell and disappeared, while
  `derive_kpu_floorplan` always reserved the full thickness in the die
  envelope -- creating a phantom moat. (Round-trip with the rasterizer
  also flagged this; the `--width` validation now catches a related
  failure mode early.)
- **`_arch_place_control`** placement: from inside the bottom-left mesh
  cell (overlapping a compute tile) to the corner gap between the IO
  ring, bottom horizontal MC strip, and left vertical MC. The control
  block is now visibly outside the mesh.

## Fixed

- **CodeRabbit (PR #152)**:
  - `cli/show_floorplan.py` -- `--width` rejects non-positive values via
    argparse type-checker.
  - `cli/show_floorplan.py` -- `--output` auto-detects `.csv` / `.md` /
    `.markdown` / `.txt`, not just `.json`. Per project CLI rules.
  - `src/graphs/hardware/silicon_floorplan.py` --
    `_classify_silicon_bin_blocks` no longer silently swallows
    `SiliconMathError`; warns to module logger so skipped blocks
    surface in stderr.
  - `src/graphs/hardware/silicon_floorplan.py` -- `_place_compute_mesh`
    and `_arch_place_checkerboard` raise `ValueError` if `arch.tiles`
    is empty (was crashing with `IndexError` on `arch.tiles[-1]`).
  - `src/graphs/hardware/silicon_floorplan.py` -- memory controllers
    anchored to die edges (just inside IO ring) instead of mesh edges;
    when extra_x / extra_y > 0, MCs no longer float inside the die.
- **Rasterizer rounding inconsistency** in `cli/show_floorplan.py`
  `_render_blocks_to_ascii`: switched both edge endpoints from a
  truncate-vs-round mix to consistent floor (`int()`). Adjacent tiles at
  the same pitch now quantize to the same row count -- previously, 8
  mesh rows rendered as 7 tiles of 2 visual rows + 1 of 3 because round
  and floor disagreed at half-pitch boundaries. Also added a min-1-cell
  rule so thin blocks (IO ring at 0.30 mm with 0.23 mm/col) don't
  collapse to zero.
- **Lint findings** (github-code-quality):
  - removed unused `_DRAM_READ_PJ_PER_BYTE` import from
    `silicon_floorplan.py` (had a `# noqa: F401` rationalization that
    was false)
  - removed unused `cell_area` local in `_arch_what_if_estimates`
  - removed unused dataclass imports
    (`ArchitecturalFloorplan`, `ArchTile`, `Floorplan`, `FloorplanBlock`,
    `TileRole`) from `tests/hardware/test_silicon_floorplan.py`
  - removed unused `ch_width` local in
    `test_mc_dimensions_independent_of_mesh_size`

## Infrastructure

  land below ERROR -- the floorplan + validator findings still surface
