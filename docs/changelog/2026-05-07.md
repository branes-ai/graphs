# Changelog 2026-05-07

V5 follow-up arc: closed all three Orin Nano calibration blockers from PR #115's
analysis, regenerated the per-hardware visualizations with the V5 analyzer, added
3-way (CPU / GPU / KPU) comparison tooling, and planned the next phase of
operator-coverage expansion.

## Added

- **Aspect-ratio-based skinny matmul detection** in `MatmulReuseModel`.
  Closes Category 2 from the Orin Nano analysis: shapes with `min(M, N) ≥ 16`
  but `max/min ≥ 32` now correctly switch to full-output K-blocking instead
  of optimal-square tiling that inflates reload counts. Memory-physics correction
  (PR #117). (`src/graphs/estimation/reuse_models.py`,
  `tests/analysis/test_reuse_models.py`,
  `docs/v5/skinny-matmul-aspect-ratio-detection.md`)
- **Per-mapper compute efficiency override**. New
  `compute_efficiency_overrides_by_op` field on `HardwareResourceModel` and
  matching `_get_compute_efficiency_override` lookup on `RooflineAnalyzer`.
  Replaces the AGX-tuned legacy curve for matmul / linear when set. Calibrated
  for Jetson Orin Nano FP16 (`{"matmul": 0.70, "linear": 0.94}`). Closes
  Category 3 from the Orin Nano analysis. PR #118.
  (`src/graphs/hardware/resource_model.py`,
  `src/graphs/estimation/roofline.py`,
  `src/graphs/hardware/models/edge/jetson_orin_nano_8gb.py`,
  `tests/analysis/test_compute_efficiency_override.py`,
  `docs/v5/jetson-compute-efficiency-derate.md`)
- **Measurement-priority sweep regime augmenter**
  (`validation/model_v4/sweeps/_augment_from_baseline.py`). For each `(hw, op)`
  with a baseline CSV, rewrites sweep regime labels from inferred measurement.
  Two safety rails: AMBIGUOUS measurements keep analytical labels; hardware
  without baselines is untouched. PR #119.
  (`tests/validation_model_v4/test_augment_from_baseline.py`,
  `docs/v5/sweep-measurement-priority-augmentation.md`)
- **3-way hardware comparison CLI** (`validation/model_v4/cli/compare_hardware.py`).
  Renders roofline / latency / energy panels overlaying multiple hardware on a
  shared shape pool. Default targets: i7-12700K + Orin Nano 8GB + KPU T64. Each
  uses native precision. PR #120.
  (`tests/validation_model_v4/test_compare_hardware.py`,
  `docs/v5/three-way-hardware-comparison.md`)
- **`--predictions-only` mode** for `visualize_baseline.py` to render hardware
  without committed V4 baselines (KPU T64, AGX, NX, H100). PR #120.
- **Plots**: standalone KPU T64 plot
  (`validation/model_v4/results/plots/kpu_t64_roofline.png`) +
  3-way comparison plot
  (`compare_i7_12700k_jetson_orin_nano_8gb_kpu_t64.png`).
- **Operator validation expansion plan**
  (`docs/v5/operator-validation-expansion-plan.md`) covering 5 phases:
  activations + SFU modeling, element-wise matrix, softmax (composite-op
  infrastructure), LayerNorm + RMSNorm, attention head as decomposed graph.
  PR #128.
- **GitHub issues**: #121 (KPU energy bug -- impossible 1175 W avg power),
  #122 (KPU scaling study -- 4-way T64/T128/T256/T768 comparison),
  #123-#127 (one per operator-expansion phase).

## Changed

- **Sweep classifier launch-bound check**
  (`validation/model_v4/sweeps/classify.py`) now uses
  `max(ideal_compute_time, ideal_memory_time)` instead of compute-only.
  Pre-fix mis-labeled memory-bound `vector_add` at large N as `LAUNCH_BOUND`
  because compute time was tiny. Six Jetson `vector_add` shapes correctly
  flipped from `launch_bound` to `dram_bound`. PR #120.
- **`_predict_latency_s` return-type unpack** in
  `validation/model_v4/cli/visualize_baseline.py`. The harness function now
  returns `(latency_s, binding_tier_name)` (V5-3b binding-tier reporting);
  visualizer was treating it as a bare float (TypeError on every run).
  Fixed; also passes `use_tier_aware_memory=True` to match V5-3b production
  default. PR #120.
- **`vector_add` added to visualizer**
  (`validation/model_v4/cli/visualize_baseline.py`). Was matmul + linear only;
  user feedback flagged that vector_add markers were missing. Added square
  marker `s` for vector_add. Also added single-int CSV shape parser for
  vector_add's `"1024"` format (not `"AxBxC"`). PR #120.
- **All six sweep JSONs re-augmented** with the new analytical classifier
  (Phase 119, 120) and measurement-priority overrides (PR #119). Net V4 PASS
  on Jetson Orin Nano went from 0/48 matmul + 0/46 linear to 10/48 + 4/46
  (tier-aware path), 11/48 + 3/46 (legacy path).
- **i7 + Orin Nano roofline plots regenerated** with the V5 analyzer
  (predicted markers were stale by 30%-2x for many shapes).
  (`validation/model_v4/results/plots/`)
- **V4 floor test thresholds** updated to reflect the new V4 PASS counts and
  per-regime tolerance shifts after PRs #119 / #120.
  (`tests/validation_model_v4/test_v4_against_baseline.py`)

## Fixed

- **GPU dispatch overhead bridge** in `RooflineAnalyzer._estimate_overhead`.
  GPUMapper had `KERNEL_LAUNCH_OVERHEAD_EDGE = 80e-6` for Jetson, but the
  analyzer used a hardcoded 5 us. Surfaced
  `kernel_launch_overhead_ns` on `HardwareResourceModel` so both can read
  the same value. Closes Category 1 from the Orin Nano analysis. PR #116
  (landed earlier in the day; included for completeness).
