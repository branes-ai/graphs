Key Operator Differences

ResNet: Standard convolutions with residual connections

  • Conv2d ‚Üí BatchNorm2d ‚Üí ReLU pattern
  • Element-wise addition for skip connections
  • Bottleneck uses 1√ó1‚Üí3√ó3‚Üí1√ó1 pattern

MobileNet: Depthwise separable convolutions

  • Depthwise Conv2d (spatial filtering)
  • Pointwise Conv2d (channel mixing)
  • Inverted residuals (expand-then-compress)

EfficientNet: Advanced attention mechanisms

  • Squeeze-and-Excitation blocks
  • Swish/SiLU activation functions
  • Stochastic depth (Dropout2d)

Activation Function Evolution

ResNet (2015):

  • ReLU: max(0, x)
  • Simple, efficient, gradient-friendly
  • Standard choice for residual networks

MobileNet (2017-2018):

  • ReLU6: min(max(0, x), 6)
  • Better for low-precision quantization
  • Bounded output for mobile deployment

EfficientNet (2019):

  • Swish/SiLU: x √ó sigmoid(x)
  • Smooth, non-monotonic activation
  • Better gradient flow and performance

MLIR Compilation Implications

Memory Access Patterns

  • ResNet: Standard conv operations with predictable memory access
  • MobileNet: Depthwise convs reduce memory bandwidth requirements
  • EfficientNet: SE blocks add global pooling operations

Parallelization Opportunities

  • ResNet: High parallelism in standard convolutions
  • MobileNet: Depthwise ops have different parallelization patterns
  • EfficientNet: SE blocks create data dependencies

Operator Fusion

  • All architectures: Conv-BN-Activation fusion opportunities
  • MobileNet: Depthwise-Pointwise fusion potential
  • EfficientNet: SE block can be fused as single operator

Quantization Considerations

  • ResNet: ReLU friendly for 8-bit quantization
  • MobileNet: ReLU6 designed for quantization
  • EfficientNet: Swish requires careful quantization handling
