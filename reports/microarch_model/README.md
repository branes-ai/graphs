# Micro-architectural Model Delivery Report

Branes-branded HTML + JSON + PowerPoint artifact describing the analytical
energy and latency models for Layers 1-7 of the 9-layer micro-architectural
hierarchy, across the locked SKU set.

Generated by `cli/microarch_validation_report.py`.

## Regenerate

```bash
# Default: all SKUs, all formats, to reports/microarch_model/latest/
./cli/microarch_validation_report.py

# Just HTML for one SKU
./cli/microarch_validation_report.py --hardware jetson_orin_agx_64gb --format html

# Just JSON
./cli/microarch_validation_report.py --format json --output reports/microarch_model/2026-04-20/

# Just PPT deck
./cli/microarch_validation_report.py --format pptx
```

The output directory contains:

```
<output>/
  index.html            # Landing page: SKU list + comparison link
  compare.html          # Cross-SKU comparison (M8 adds interactive charts)
  hardware/             # Per-SKU layer panels (Layers 1-7)
    <sku>.html
  data/                 # JSON data contract per SKU
    <sku>.json
  microarch_deck.pptx   # One engineering-audience deck
```

## Confidence ladder

Every metric in every layer panel ships with one of four tags. The higher
tags require stronger evidence.

| Tag | Meaning |
|-----|---------|
| `CALIBRATED`   | Fit against measurement on the specific SKU at a known thermal point. Highest confidence. |
| `INTERPOLATED` | Fit on a sibling SKU in the same family; value extrapolated. |
| `THEORETICAL` | Derived from vendor datasheets, published papers, or first-principles analytical model. |
| `UNKNOWN`     | Not yet modeled. |

During the M1-M7 delivery push all entries ship as `THEORETICAL` or
`INTERPOLATED`. `CALIBRATED` lands only after the follow-on validation
phase (measurement-driven calibration) completes.

## Hierarchy and validation phase

Layers 1-7 are populated in this epic:

1. ALU / MAC / Tensor Core
2. Register file
3. L1 cache / scratchpad
4. L2 cache
5. L3 / last-level cache
6. SoC data movement (on-chip fabric)
7. External memory

Layers 8 (intra-server fabric) and 9 (cluster interconnect) are defined
in the schema but panels remain empty for this epic. They are addressed
in a separate follow-on.

## Handoff notes for later validation

This report is produced by analytical models only; no measurement is
interleaved with milestones M0-M8. After the delivery epic ships, the
bottom-up microbenchmark plan
(`docs/plans/bottom-up-microbenchmark-plan.md`) re-activates and each
layer's `THEORETICAL` entries graduate to `CALIBRATED` in place as
measurements land. The JSON data contract in `data/` is the stable
interface the measurement pass writes to.
